# Week5 統合ダッシュボード実装仕様

## 1. 目的
- Week2〜Week4で分離していた体験を単一URLで完結させる。
- Big4向けに「3秒把握、1分説明、5分提案」を満たす。

## 2. 画面情報設計（UI Flow）
- Header
  - デモ目的、現在フェーズ、選択候補者
- Phase Stepper
  - `PreCheck -> MeetingLive -> FinalizeLock -> PostAnalytics -> AutoReport`
- Main Tabs
  - 会議ライブ
  - 確定・監査
  - 分析・レポート
  - AI判定基準
- Side Panel
  - 候補者サマリ
  - 重要アラート
  - 次アクション

## 3. integratedState スキーマ
```js
{
  masterData: {
    candidates, p0Records, evidenceCatalog, meetingScenarios, nudgeRules,
    finalDecisionRecords, auditSeedEvents, analyticsSeed, reportTemplates
  },
  session: { selectedCandidateId, currentPhase, activeTab, lastActionAt },
  meeting: { eventLog, nudgeLog, speakerDurations, timeline, nineBoxGapDraft },
  decision: { unlockApproved, unlockRequestDraft },
  audit: { events, tamperFlags, hashCheckResults },
  analytics: { snapshot, kpis, generatedReport },
  ui: { drilldownTarget, toast, loading }
}
```

## 4. イベント連携
- 共通ディスパッチャ: `dispatchEvent(type, payload)`
- 主要イベント:
  - `candidate_selected`
  - `meeting_started`, `speaker_turn_changed`, `nudge_shown`, `nudge_accepted`, `nudge_dismissed`, `ninebox_gap_saved`
  - `finalize`, `unlock_request`, `unlock_approved`, `edit_after_unlock`
  - `analytics_recomputed`, `report_generated`, `drilldown_opened`
- 連鎖:
  - `finalize/edit_after_unlock` -> 監査イベント追加 -> hash再計算 -> `analytics_recomputed`

## 5. 不変性ルール（改ざん防止演出）
- `finalizedFlag=true` かつ `unlockApproved=false` の場合:
  - 判定、理由、タグ、決定者入力を編集不可
- `unlock_approved` 後のみ一時編集可能
- 編集後は `lockVersion` 更新と `recordHash` 再計算を必須

## 6. 説明可能性ルール
- KPIカード、監査表示、レポートに必ず根拠参照を表示
- AI判定基準タブに以下を常時表示
  - 入力情報（定量、定性、文脈タグ）
  - 判定プロセス（データ品質 -> 根拠構造化 -> 判定 -> 人間補正）
  - 「AIは最終決裁しない」ガードレール

## 7. Big4向けUXガイド
- 3秒把握:
  - KPI 5種、監査健全性、現在フェーズを上段固定
- 1分説明:
  - 各タブに「経営向け要約（3行）」を配置
- 5分提案:
  - レポートに「改善アクション（担当/期限/期待効果）」を出力

## 8. 受け入れ基準
- 単一URLで理想シナリオを完走できる
- Finalize後の編集禁止と例外解除フローが確認できる
- KPI/監査/レポートが連動更新される
